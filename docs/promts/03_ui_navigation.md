# Промт 3: Слой UI-интерфейса, индикатор шагов и навигация

## Задача
Реализовать фиксированный UI-слой (Header, меню навигации, прогресс-бар и индикатор текущего шага `[STEP.XX/05]`) поверх скроллируемых панелей с поддержкой плавной программной прокрутки.

## Инструкции по реализации

### 1. Верстка фиксированного интерфейса
Создайте UI-компоненты поверх `#viewport` (с `position: fixed` или абсолютным позиционированием и высоким `z-index`):
*   **Header**:
    *   Логотип или имя разработчика слева.
    *   Ссылки на разделы справа (`Intro`, `Education`, `Experience`, `About`, `Contacts`).
*   **Scroll Indicator (Прогресс-бар)**:
    *   Тонкая горизонтальная линия (`height: 1px` или `2px`) внизу страницы, ширина которой анимируется от 0% до 100% ширины экрана на основе общего прогресса скролла.
*   **Индикатор шага**:
    *   Лейбл в углу экрана с моноширинным шрифтом JetBrains Mono, отображающий текущий шаг в формате `[STEP.01/05]`, `[STEP.02/05]` и т.д.

### 2. Логика плавной навигации
Добавьте обработчики кликов на ссылки в Header:
*   При клике на ссылку вычислите целевую высоту прокрутки окна для данной секции:
    *   `targetScrollY = index * window.innerHeight` (так как высота скролл-контейнера `h-[500vh]`, то каждая панель занимает ровно `100vh` по высоте прокрутки).
*   Запустите плавную прокрутку окна к этой координате с помощью GSAP:
    *   `gsap.to(window, { scrollTo: targetScrollY, duration: 1.0, ease: 'power2.out' })`
    *   *Примечание*: Для работы плагина ScrollToPlugin убедитесь, что он импортирован и зарегистрирован: `gsap.registerPlugin(ScrollToPlugin)`.

### 3. Доступность и Reduced Motion
*   Проверьте медиа-запрос `prefers-reduced-motion: reduce`.
*   Если пользователь предпочитает уменьшенное движение, при клике на ссылки меню скролл должен осуществляться мгновенно:
    *   `gsap.to(window, { scrollTo: targetScrollY, duration: 0 })` или через `window.scrollTo(0, targetScrollY)`.
*   Настройте порядок табации элементов (`tabindex`) слева направо. При переходе фокуса на элементы за пределами экрана прокручивайте viewport к соответствующей секции.

## Критерии приемки
1. Индикаторы шага (например, `[STEP.03/05]`) и ширина линии прогресса внизу обновляются синхронно при прокрутке.
2. Клик по ссылкам меню плавно перемещает страницу к нужной панели.
3. При включении режима "уменьшение движения" в системе/браузере, переходы по ссылкам происходят мгновенно, без плавной анимации.

# Промт 5: GPU Шейдеры (GLSL) и морфинг 5 форм частиц

## Задача
Написать GLSL вершинный и фрагментный шейдеры для частиц, сгенерировать 5 наборов координат для секций портфолио и реализовать логику перестроения (морфинга) на базе GSAP.

## Инструкции по реализации

### 1. Вершинный шейдер (Vertex Shader)
Напишите шейдер в `ShaderMaterial`, содержащий:
*   Юниформы: `uTime` (время), `uMorphProgress` (прогресс морфинга от 0 до 1), `uPointer` (координаты курсора мыши в world space).
*   Атрибуты: `position` (исходные координаты), `aTargetPosition` (координаты целевой формы).
*   Логика:
    *   Интерполяция текущей позиции вершины: `vec3 mixedPosition = mix(position, aTargetPosition, uMorphProgress);`.
    *   Эффект отталкивания от курсора: вычислите расстояние от вершины до `uPointer` в плоскости XY. Если оно меньше 2.0 единиц, смещайте вершину радиально в сторону от курсора с силой, обратно пропорциональной расстоянию: `modelPosition.xy += normalize(modelPosition.xy - uPointer) * force * 0.5;`.
    *   Размер точки с перспективным делением: `gl_PointSize = 2.0 * (300.0 / -viewPosition.z);`.

### 2. Фрагментный шейдер (Fragment Shader)
*   Вычислите расстояние от центра точки `distance(gl_PointCoord, vec2(0.5))`.
*   Отсекайте внешнюю часть круга: `if (distanceToCenter > 0.5) discard;`.
*   Сглаживайте края точки с помощью `smoothstep`: `strength = 1.0 - smoothstep(0.4, 0.5, distanceToCenter);`.
*   Задайте цвет частиц: платиновый `#DCDCDC` (`vec3(0.86, 0.86, 0.86)`), opacity `strength * 0.8`.

### 3. Генерация 5 геометрических форм
На основе `particlesCount` сгенерируйте 5 Float32Array массивов:
1.  **Секция 0: Intro (Grid)**: Горизонтальная плоскость по сетке с легким синусоидальным шумом высоты по оси Z.
2.  **Секция 1: Education (Rings)**: Набор концентрических окружностей разного радиуса, медленно вращающихся вокруг центра.
3.  **Секция 2: Experience (Timeline Axis)**: Тонкая горизонтальная линия по центру экрана с короткими вертикальными черточками (отметками дат).
4.  **Секция 3: About (Sphere)**: Равномерно распределенные точки на поверхности 3D-сферы (например, по алгоритму Фибоначчи).
5.  **Секция 4: Contacts (Cloud)**: Хаотичное облако случайных точек с плотностью, убывающей к краям экрана (броуновское движение частиц).

### 4. Логика морфинга (`morphTo`)
Реализуйте метод `morphTo(sectionIndex, progress?)` в `BackgroundGraphicsService`:
*   Запишите массив целевых координат в атрибут геометрии `aTargetPosition` и установите флаг `needsUpdate = true`.
*   Если передан аргумент `progress` (скруббинг скролла):
    *   Привяжите `uMorphProgress.value` напрямую к прогрессу.
    *   При достижении `progress === 1.0` выполните своп буферов: скопируйте массив `aTargetPosition` в `position` и сбросьте `uMorphProgress.value = 0` (метод `swapBuffers()`).
*   Если аргумент `progress` опущен (триггерный переход):
    *   Запустите анимацию GSAP: `gsap.to(uniforms.uMorphProgress, { value: 1, duration: prefersReducedMotion ? 0 : 1.5, ease: 'power2.inOut', onComplete: () => this.swapBuffers() })`.

## Критерии приемки
1. Частицы рендерятся в виде аккуратных сглаженных кругов платинового цвета.
2. При смене секции частицы плавно перестраиваются (морфинг) из одной геометрической формы в другую.
3. Точки отталкиваются от курсора мыши и плавно возвращаются обратно (пружинящий эффект).
4. В режиме `prefers-reduced-motion` морфинг происходит мгновенно (duration: 0).

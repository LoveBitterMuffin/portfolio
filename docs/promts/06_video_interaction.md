# Промт 6: Интерактивное видео-силуэт и параллакс (VideoInteractionService)

## Задача
Интегрировать видеофайл с силуэтом человека, реализовать эффект скруббинга (перемотки кадров) от движения мыши и настроить адаптивный параллакс-эффект смещения контейнера видео.

## Контекст и файлы
*   **Спецификация видео-сервиса**: [design.md](file:///c:/mp/portfolio/docs/services/video-interaction/design.md)
*   **Правила разработки (Video)**: [agents.md](file:///c:/mp/portfolio/docs/services/video-interaction/agents.md)
*   **Файл видео**: `/public/lbm.mp4`

## Инструкции по реализации

### 1. Верстка видео-контейнера и маскирование
*   Разместите элемент `<video>` в фиксированном или абсолютно спозиционированном контейнере поверх фонового канваса с частицами.
*   Установите для видео атрибуты: `src="/lbm.mp4"`, `muted`, `playsInline`, `preload="auto"`.
*   Примените к видео-контейнеру CSS-свойство `mix-blend-mode: multiply` (`mix-blend-multiply` в Tailwind). Это скроет белый фон видеоролика, сделав его прозрачным, в то время как темный силуэт человека останется непрозрачным и будет перекрывать точки фоновой сцены.
*   **Критично**: Задайте для видео-контейнера класс `pointer-events-none` (свойство `pointer-events: none`). Это необходимо, чтобы клики и движения мыши беспрепятственно проходили сквозь видеоролик на Three.js сцену.
*   **Адаптивность**: Задайте ширину контейнера в зависимости от экрана: `w-[75vw]` на мобильных и `w-[38vw]` на десктопах (начиная с брейкпоинта `md:`).

### 2. Реализация класса `VideoInteractionService`
Создайте файл `src/services/VideoInteractionService.ts` на TypeScript:
*   **Скруббинг видеоролика (перемотка)**:
    *   При движении мыши в пределах Intro-секции рассчитайте относительную горизонтальную координату курсора: `progress = clientX / window.innerWidth`.
    *   Проецируйте её на длительность видео: `targetTime = progress * video.duration`.
    *   Анимируйте `currentTime` видеоролика через GSAP сглаживание:
        ```typescript
        gsap.to(this.playheadState, {
          currentTime: targetTime,
          duration: 0.6,
          ease: 'power1.out',
          overwrite: 'auto',
          onUpdate: () => { this.video.currentTime = this.playheadState.currentTime; }
        });
        ```
    *   *Важно*: Скруббинг видео — тяжелая операция для CPU/GPU. Задайте троттлинг обновления `video.currentTime` с частотой не более одного раза в `33ms` (примерно 30 кадров/с), сравнивая метки `performance.now()`.
*   **Параллакс видео-контейнера**:
    *   При движении мыши плавно смещайте контейнер видеоролика в противоположную от курсора сторону с коэффициентом `PARALLAX_STRENGTH = 24`.
    *   Используйте GSAP для интерполяции координат смещения:
        ```typescript
        gsap.to(this.container, { x: offsetX, y: offsetY, duration: 1.2, ease: 'power2.out', overwrite: 'auto' });
        ```

### 3. Доступность (Reduced Motion) и мобильная деградация
*   В режиме `prefers-reduced-motion: reduce`:
    *   Отключите параллакс-эффект контейнера видеоролика (`PARALLAX_STRENGTH = 0`).
    *   Скруббинг видеоролика (`currentTime`) должен оставаться активным, так как это прямое интерактивное действие пользователя.
*   На мобильных устройствах:
    *   Полностью отключите параллакс-смещение (тач-интерфейс не обеспечивает плавности).
    *   Увеличьте интервал троттлинга скруббинга видеоролика `throttleMs` с `33ms` до `66ms` или `100ms` для снижения нагрузки на декодер мобильного GPU.
    *   В случае низкого FPS предусмотрите скрытие видео и показ статичного силуэта `.png` с `mix-blend-mode: multiply`.

## Критерии приемки
1. Видеоролик корректно проигрывается вперед/назад вслед за движением мыши по горизонтали в первой секции.
2. Белый фон видеоролика скрыт, силуэт человека перекрывает платиновые точки Three.js фона.
3. События клика и движения мыши проходят сквозь силуэт на фоновую WebGL-сцену (точки реагируют под силуэтом).
4. На мобильных устройствах и в режиме reduced motion параллакс смещения контейнера видео отключается.

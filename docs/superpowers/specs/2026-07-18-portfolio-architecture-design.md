## Резюме архитектурного решения

Мы выбрали **Централизованную API-ориентированную архитектуру** (Approach 1), которая обеспечивает наилучшую производительность рендеринга и плавность анимаций при частоте кадров 60+ FPS.

1.  **Управление состоянием**: Мы отказались от использования реактивного состояния React для передачи высокочастотных обновлений (движение мыши, скролл). Данные передаются напрямую в сервисы через Refs и вызовы методов инстансов.
2.  **Эстетика**: Выбран **белый цифровой минимализм (Digital Blueprint)**. Стерильный белый фон, разреженная сетка из микрочастиц Three.js (1-2px), изящный тонкорамочный интерфейс и моноширинная техническая типографика.
3.  **Интерактивный контент**: Данные загружаются из внешнего файла `docs/content.md`. Сетка частиц Three.js морфится в различные 3D-формы при горизонтальном перелистывании секций:
    *   *Intro* — плоская сетка, обтекающая силуэт человека на видео.
    *   *Education* — аккуратные кольцевые/структурированные 3D-формы.
    *   *Experience* — линейные таймлайн-оси.
    *   *About / Tech Stack* — вращающаяся 3D-сфера.
    *   *Contacts* — разреженное облако, притягивающееся к кнопкам.

## План верификации производительности

### Тестирование CPU/GPU в Chrome DevTools
1.  Запустить приложение в режиме сборки `npm run build && npm run start`.
2.  Открыть вкладку Performance в Chrome DevTools и записать 10 секунд активного скролла и движения мыши.
3.  Проверить отсутствие "длинных задач" (Long Tasks > 50ms) и убедиться, что график частоты кадров стабильно держится на уровне целевого FPS.
4.  Вкладка Memory: Убедиться, что после 5 полных циклов прокрутки туда-обратно потребление памяти (JS Heap) возвращается к базовому значению (отсутствие утечек памяти в Three.js и GSAP).
